# BKS Promotions & Offers Skill

## 0. Brand constraint (non-negotiable)

BKS never runs permanent sales or default discount states.
A discount is an editorial event — it has a name, a window, a story.
"50% OFF EVERYTHING" is never a BKS message.

---

## 1. Approved discount types

| Type | Mechanics | Max discount | Gate |
|------|-----------|-------------|------|
| **Drop Launch** | Code at checkout, 72h window | 15% | Public |
| **Member Early Access** | Auto-applied for Brass+, pre-drop | 10% | Brass+ |
| **Tier Milestone** | One-time reward on tier upgrade | 15% | Auto on upgrade |
| **Bundle** | 2+ items from same collection | 10% | No gate |
| **Reactivation** | 30-day inactive members | 12% | Iron+ |
| **Seasonal Clearance** | End-of-series only | 20% max | Public, labeled |
| **Press / Collab** | Manual code for press/partners | Flat €20 or 15% | Manual |

**Never:** permanent discount banners, default "sale" state, >20% off on core catalog.

---

## 2. Discount code naming convention

Format: `BKS-[TYPE]-[YYYYMM]` or `BKS-[COLLECTION]-[EVENT]`

Examples:
- `BKS-DROP07-202607` → Drop 07 launch code
- `BKS-BRASS-EARLY` → Brass tier early access
- `BKS-GOLD-VIPJUN` → Gold VIP June event
- `BKS-HOURS-LAUNCH` → BKS Hours relaunch

**Never:** random strings, confusing codes, expiration dates in the code name.

## 3. Margin floor during promotions

No product goes below 40% margin after discount.

Quick check:
```
discounted_retail = original_price × (1 - discount_pct/100)
net_revenue = discounted_retail × 0.971 - 0.30
margin = (net_revenue - total_cost) / net_revenue
→ Must be ≥ 40%
```

Example: Racerback Dress at $65, 15% off = $55.25.
  Net = $55.25 × 0.971 - 0.30 = $53.40. Cost = $32.
  Margin = ($53.40 - $32) / $53.40 = 40.1% — just above floor. ✓

Never apply 15% to products already near minimum price (dresses, tees, slippers).
Apply 15% only to puffers, windbreakers, sneakers, travel bags.

## 4. Drop launch sequence

```
T-72h: Email Brass+ "Early Access confirmed — code expires in 72h"
       Code: 10% off, valid 72h, Brass+ auto-applied
T-48h: Push to wishlist owners "Your wishlist piece drops soon"
T-24h: Silver+ preview link (private URL, no code needed)
T=0:   Public drop live — landing page live, Instagram + Telegram push
T+12h: "Only X slots remaining" signal (if >30% of variants ordered)
T+48h: Drop closes — code expires — archive entry created
```

## 5. Tier milestone rewards (automatic via Shopify Flow)

| Trigger | Reward | Code validity |
|---------|--------|--------------|
| First order (Lead→Iron) | 10% on next order | 30 days |
| 3rd order (Iron→Brass) | 15% on next order + Try-On access | 30 days |
| 6th order (Brass→Silver) | 12% on next order | 60 days |
| 11th order (Silver→Gold) | 20% on next collection drop (VIP) | 90 days |

Code naming: `BKS-TIER-[CUSTOMER_FIRST_6]-[TIER]` (auto-generated by Flow)

## 6. Seasonal events calendar

| Event | Window | Discount type | Max |
|-------|--------|--------------|-----|
| Summer Drop | June–July | Drop Launch | 10% |
| Back to Collection | September | Relaunch | 12% |
| Black Friday Alternative | November | "Studio Week" editorial | 15% |
| Year-End | December | Member exclusive | 20% (Gold only) |

**Note:** BKS does NOT call it "Black Friday". Use "BKS Studio Week" or "November Edit".

## 7. Shopify implementation

- **Discount codes**: Admin → Discounts → Create
- **Automatic discounts**: Admin → Discounts → Create automatic
- **Flow triggers**: Shop → Flow → "Order created" → check tier tag → apply discount
- **Brass+ auto-apply**: Use customer tag `tier:brass` OR `tier:silver` OR `tier:gold` → automatic discount
- **Announcement bar**: Use Essential Announcer app, set duration = event window only

---

## 8. What NOT to do

- No countdown timers ("Offer ends in 03:42:11")
- No "SALE" in product titles — keep editorial names clean
- No stacking promotions without explicit approval
- No "limited stock" messages when stock is actually unlimited (POD)
- No urgency language: "Buy now before it's gone!" — BKS is made-to-order, it's always available
  EXCEPTION: limited colorways = genuinely limited, can say "Last colorways available"
